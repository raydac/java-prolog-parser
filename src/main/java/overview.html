The Java Prolog Parser was developed as a part of the Prol engine (it is an embeddable Prolog Java engine) but lately it was distinguished as a free-standing open source project to allow for everyone to develop own Prolog engines in Java.
    The parser allows to parse prolog sources written in the Edinburgh style (pay your attention, it supports only '%' comments). It can use below Java objects as char data sources:
  • java.lang.String
  • java.io.InputStream
  • java.nio.channels.ReadableByteChannel
  • java.io.Reader
    The parser supports two kinds of numbers - integer and float, integer numbers are based on the Long Java type and float numbers are based on the Double Java type.

The parser supports below escape symbols in strings:
  • \a (code 7)
  • \b (code 8)
  • \n (the next line symbol)
  • \r (the caret return symbol)
  • \e (code 27)
  • \t (tabulation)
  • \s (code 32)
  • \v (code 11)
  • \\ (symbol '\')
  • \' (symbol ''')
  • \xAA (hexadecimal 16 bit char code)
  • \uAAAA (hexadecimal 32 bit char code)
Also you can broke a string for a few strings if to place the '\' before the next line.
